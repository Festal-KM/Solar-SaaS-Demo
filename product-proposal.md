# 太陽光卸・二次店営業管理システム 仕様書

## 1. システム概要

本システムは、太陽光パネルの卸事業を行う卸業者と、営業活動を担う二次店事業者が共同で利用する業務管理ツールである。

卸業者はメーカーから太陽光パネルを仕入れ、自社利益を上乗せして販売する。販売活動は、卸業者の自社リソースで実施する場合、二次店に委託する場合、または自社と二次店が共同で実施する場合がある。

営業活動は主に以下のチャネルで行う。

* 催事営業
* 飛び込み営業
* テレコール営業
* 紹介営業
* その他営業活動

催事営業では、カインズ・コメリなどの店舗で土日を中心にイベントを実施する。イベント実施前には、卸業者が場所提供元とコミュニケーションを行い、実施可能な場所・日程を確定する。その後、イベント候補としてシステムに登録する。

イベント候補は二次店に共有され、二次店は月単位で希望する店舗を複数選択して回答する。卸業者は二次店の希望を確認したうえで、最終的に各イベントの開催体制を決定する。

開催体制は以下のいずれかとする。

* 自社開催
* 二次店開催
* 自社・二次店共同開催
* 中止

自社開催または共同開催の場合は、自社要員のシフト管理が必要となる。二次店開催または共同開催の場合は、担当二次店を設定する。

本システムでは、場所提供元とのコミュニケーション管理、イベント候補管理、二次店希望収集、開催体制管理、自社要員シフト管理、イベント報告、顧客・アポ管理、マエカク、マエカク結果連絡、商談・クロージング、契約、施工、補助金申請、商品・価格管理、粗利管理、月次報告、成績管理、インセンティブ計算までを一元管理する。

## 2. ビジネスモデル

### 2.1 登場事業者

| 事業者     | 役割                                          |
| ------- | ------------------------------------------- |
| 卸業者     | 太陽光パネルをメーカーから仕入れ、販売・営業管理・施工管理・手数料管理を行う      |
| 卸業者自社要員 | 自社開催または共同開催イベントに参加し、現場対応・アポ獲得・報告を行う         |
| 二次店     | 卸業者の営業活動を支援する。催事、アポ獲得、初回訪問、商談・クロージングなどを担当する |
| メーカー    | 卸業者に太陽光パネル等の商品を供給する                         |
| 施工業者    | 契約後の施工を担当する                                 |
| 場所提供元   | 催事実施場所を提供する店舗・施設等                           |

---

### 2.2 イベント開催形態

| 開催形態       | 内容                        |
| ---------- | ------------------------- |
| 自社開催       | 卸業者の自社要員のみでイベントを実施する      |
| 二次店開催      | 二次店のみでイベントを実施する           |
| 自社・二次店共同開催 | 卸業者の自社要員と二次店が共同でイベントを実施する |
| 中止         | イベントを実施しない                |

---

### 2.3 二次店の業務スコープ

二次店の担当範囲は会社・案件ごとに異なる。

本システムでは、二次店スコープを以下の通り定義する。

| スコープ        | 内容                                                |
| ----------- | ------------------------------------------------- |
| アポ獲得まで      | 顧客情報の取得および訪問アポイントの獲得までを担当する。トスアップのみのケースも本スコープに含める |
| 初回訪問まで      | 顧客宅への初回訪問まで担当する                                   |
| 商談・クロージングまで | 商品説明、提案、見積説明、契約獲得までを一体で担当する                       |

以下は二次店の業務スコープとしては扱わない。

| 除外項目     | 理由                          |
| -------- | --------------------------- |
| マエカク     | 卸業者側のコール部隊が実施する前提           |
| 商談のみ     | 商談とクロージングは分離せず、一体のスコープとして扱う |
| クロージングのみ | 商談・クロージングを分けず、一体のスコープとして扱う  |
| 契約後フォロー  | 二次店スコープとしては扱わない             |

### 2.4 商品・価格・粗利の考え方

卸業者はメーカーから太陽光パネル等の商品を仕入れ、二次店に対して卸値を設定する。

本システムでは、商品ごとに以下の価格情報を管理する。

| 項目     | 内容                            |
| ------ | ----------------------------- |
| 仕入値    | 卸業者がメーカーから仕入れる価格              |
| 二次店卸値  | 卸業者が二次店に提示する基準価格              |
| 希望販売価格 | 顧客への販売価格の基準値                  |
| 実販売価格  | 契約時点で顧客に提示・契約した価格             |
| 粗利     | 実販売価格から仕入値、施工費、その他原価等を差し引いた利益 |

商品マスタ上では、仕入値・二次店卸値・希望販売価格の基準値を管理する。

契約時には、契約時点の価格情報を契約明細に保持する。これにより、商品マスタの価格が変更されても、過去契約の粗利・インセンティブ計算結果が変わらないようにする。

---

### 2.5 インセンティブの基本考え方

二次店へのインセンティブは、売上ではなく粗利を基準にパーセンテージで計算する。

基本計算式は以下とする。

```text
インセンティブ額 = インセンティブ対象粗利 × インセンティブ率
```

インセンティブ対象粗利は、運用ルールに応じて以下のいずれかを選択できるものとする。

| インセンティブ対象粗利 | 内容                                |
| ----------- | --------------------------------- |
| 案件粗利        | 実販売価格から仕入値、施工費、その他原価、値引き額を差し引いた粗利 |
| 卸粗利         | 二次店卸値合計から商品仕入値合計を差し引いた粗利          |
| 手動指定粗利      | 管理者が個別に調整した粗利                     |

案件粗利の基本計算式は以下。

```text
案件粗利 = 実販売価格 - 商品仕入値合計 - 施工費 - その他原価 - 値引き額
```

卸粗利の基本計算式は以下。

```text
卸粗利 = 二次店卸値合計 - 商品仕入値合計
```

粗利が0円以下の場合、インセンティブは原則0円とする。

---

### 2.6 インセンティブ対象

| 対象         | インセンティブ対象                |
| ---------- | ------------------------ |
| 自社開催       | 二次店インセンティブ対象外            |
| 二次店開催      | 担当二次店にインセンティブを設定可能       |
| 自社・二次店共同開催 | 二次店の担当範囲に応じてインセンティブを設定可能 |

インセンティブは、原則として契約成立後に計算対象とする。

キャンセル案件はインセンティブ対象外とする。

## 3. 業務フロー

### 3.1 催事営業の全体フロー

1. 卸業者が場所提供元とコミュニケーションを行う
2. イベント実施可能な場所・日程を調整する
3. 場所提供元との条件を確認する

   * 固定費型
   * 成果報酬型
   * その他条件
4. 場所・日程が確定する
5. 卸業者がイベント候補としてシステムに登録する
6. 卸業者が二次店にイベント開催候補を共有する

   * 二次店に共有するのは原則として場所と日にちのみ
7. 二次店は月単位で希望する店舗を複数選択して提出する
8. 卸業者が二次店の希望内容を確認する
9. 卸業者がイベント候補ごとに開催体制を決定する

   * 自社開催
   * 二次店開催
   * 自社・二次店共同開催
   * 中止
10. 二次店開催または共同開催の場合、卸業者が担当二次店を決定する
11. 自社開催または共同開催の場合、卸業者が自社要員のシフトを割り当てる
12. 開催体制決定後、該当する二次店・自社要員に通知する
13. イベントを実施する
14. 開始報告・終了報告・成果報告を行う
15. アポを獲得した顧客情報を登録する
16. 卸業者のコール部隊がマエカクを実施する
17. 卸業者がマエカク結果を二次店へ連絡する
18. 初回訪問を行う
19. 商談・クロージングを行う
20. 契約が成立する
21. 契約時点の商品・価格情報を契約明細として登録する
22. 契約単位で粗利を計算する
23. 施工業者へ施工を委託する
24. 施工進捗を管理する
25. 補助金等の申請状況を管理する
26. 月次報告を作成・確認する
27. 成績・インセンティブを確定する

---

### 3.2 場所取り・場所提供元対応フロー

イベント候補を登録する前に、卸業者は場所提供元との調整を行う。

#### 業務内容

* 場所提供元への連絡
* 実施可能日程の確認
* 実施条件の確認
* 費用条件の確認
* 成果報酬条件の確認
* 実施可否の確定
* イベント候補として登録

#### 管理したい情報

* 場所提供元名
* 担当者名
* 連絡先
* 場所名
* 住所
* 実施候補日
* 確定日
* 契約条件

  * 固定費型
  * 成果報酬型
  * その他
* 固定費金額
* 成果報酬率
* 条件メモ
* ステータス

#### ステータス

* 未連絡
* 調整中
* 実施可
* 実施不可
* 条件確認中
* 確定
* 中止

### 3.3 イベント候補から開催体制決定までの流れ

イベントは、以下の4段階で管理する。

#### 第1段階：場所提供元との調整

卸業者が場所提供元とコミュニケーションを行い、実施可能な場所・日程を確定する。

#### 第2段階：イベント候補登録

確定した場所・日程を、卸業者がイベント候補として登録する。

#### 第3段階：二次店希望収集

二次店に対して、イベント候補のうち以下のみを共有する。

* 場所
* 日にち
* 回答期限

二次店は、対象月に実施したい店舗を複数選択して希望提出する。

#### 第4段階：卸業者による開催体制決定

卸業者が二次店の希望を確認したうえで、各イベント候補について最終的な開催体制を決定する。

このとき、どの業者がどの店舗で実施するかは卸業者が決定する。

---

### 3.4 マエカク後の結果連絡フロー

マエカクは卸業者側のコール部隊が実施する。

マエカク実施後、卸業者は関係する二次店に結果を連絡する。

#### 連絡対象

* 当該アポを獲得した二次店
* 当該顧客の初回訪問または商談・クロージングを担当する二次店

#### 連絡内容

* マエカク実施日時
* コール結果
* 訪問了承有無
* 確認済み訪問日時
* 確認済み訪問場所
* キャンセル有無
* 日程変更有無
* 注意事項
* 次回アクション

### 6.16 F-016 粗利計算管理

契約単位で売上、仕入値、施工費、その他原価を管理し、粗利を算出する機能。

#### 管理項目

* 契約
* 顧客
* 実販売価格
* 商品仕入値合計
* 二次店卸値合計
* 施工費
* その他原価
* 値引き額
* 案件粗利
* 粗利率
* インセンティブ対象粗利
* インセンティブ対象粗利の種類
* 備考

#### 計算式

```text
商品仕入値合計 = 契約明細ごとの仕入値 × 数量 の合計
```

```text
二次店卸値合計 = 契約明細ごとの二次店卸値 × 数量 の合計
```

```text
案件粗利 = 実販売価格 - 商品仕入値合計 - 施工費 - その他原価 - 値引き額
```

```text
粗利率 = 案件粗利 ÷ 実販売価格
```

```text
インセンティブ対象粗利 = 案件粗利
```

または、運用ルールに応じて以下の計算式も選択可能とする。

```text
インセンティブ対象粗利 = 二次店卸値合計 - 商品仕入値合計
```

#### 要件

* 契約ごとに粗利を自動計算する
* 契約明細に商品・数量・仕入値・二次店卸値を保持する
* 商品マスタ変更後も、過去契約の粗利が変わらないようにする
* 施工費・その他原価を加味して粗利を算出できる
* 値引き額を粗利計算に反映できる
* インセンティブ対象粗利を手動調整できる
* 手動調整した場合は変更履歴を残す

---

### 6.17 F-017 インセンティブ計算

二次店へのインセンティブを、売上ではなく粗利を基準に計算する機能。

#### 基本計算式

```text
インセンティブ額 = インセンティブ対象粗利 × インセンティブ率
```

#### 例

```text
実販売価格：2,000,000円
仕入値合計：1,200,000円
施工費：300,000円
その他原価：100,000円
案件粗利：400,000円
インセンティブ率：30%

インセンティブ額 = 400,000円 × 30% = 120,000円
```

#### 要件

* インセンティブは粗利ベースで計算する
* 売上金額に対するパーセンテージでは計算しない
* 二次店スコープごとにインセンティブ率を設定できる
* 契約ごとにインセンティブ対象粗利を保持する
* インセンティブ率は契約時点の値を保持する
* 粗利が0円以下の場合、インセンティブは原則0円とする
* キャンセル案件はインセンティブ対象外とする
* 自社開催のみの案件は二次店インセンティブ対象外とする
* 二次店開催または自社・二次店共同開催案件のみ、二次店インセンティブ計算対象とする

## 9. 通知要件

### 9.1 卸業者向け通知

以下のタイミングで通知する。

* 二次店が希望店舗を提出したとき
* 希望回答期限を過ぎても未回答の二次店があるとき
* 開催体制未決定のイベント候補があるとき
* 自社開催イベントのシフト割当が不足しているとき
* 二次店がイベント開始報告を登録したとき
* 自社要員がイベント開始報告を登録したとき
* イベント終了報告が登録されたとき
* イベント成果報告が登録されたとき
* 新規アポ顧客が登録されたとき
* マエカク未対応のアポがあるとき
* マエカク結果が未連絡の二次店があるとき
* 二次店が商談・クロージング報告を登録したとき
* 商談・クロージング報告で契約ステータスになったとき
* 月次報告が提出されたとき
* 粗利未計算の契約があるとき
* インセンティブ未計算の契約があるとき
* 施工予定日が近づいたとき
* 申請期限が近づいたとき

---

### 9.2 二次店向け通知

以下のタイミングで通知する。

* 新しいイベント候補が公開されたとき
* 希望店舗回答期限が近づいたとき
* 二次店開催または共同開催として担当イベントが決定したとき
* イベント前日になったとき
* 報告未提出のイベントがあるとき
* マエカク結果が共有されたとき
* 自社関与案件が契約になったとき
* 月次報告の確認依頼があるとき
* インセンティブが確定したとき

* イベント開始報告が未提出のとき

## 10. バリデーション要件

### 10.1 場所提供元対応

* 場所提供元名は必須
* 場所名は必須
* 実施可否が確定するまではイベント候補化しない
* 契約条件が固定費型の場合、固定費金額を入力する
* 契約条件が成果報酬型の場合、成果報酬率を入力する

---

### 10.2 イベント候補登録

* 対象年月は必須
* 実施予定日は必須
* 場所名は必須
* 場所提供元対応情報と紐づけ可能とする

---

### 10.3 希望店舗回答

* 対象年月は必須
* 希望店舗は1件以上必須
* 回答期限後は二次店側から編集不可
* 同一二次店が同一イベント候補に重複回答できない

---

### 10.4 開催体制決定

* 開催体制は必須
* 二次店開催の場合、担当二次店は必須
* 共同開催の場合、担当二次店と自社要員シフトの両方を設定する
* 自社開催または共同開催の場合、必要人数を設定できる
* 中止の場合、イベント作成は不要
* 開催体制変更時は履歴を残す

---

### 10.5 自社シフト割当

* 自社開催または共同開催イベントには1名以上のシフト割当を推奨する
* 必要人数に対して割当人数が不足している場合、警告を表示する
* 勤務開始予定時刻は必須
* 勤務終了予定時刻は必須
* 終了時刻は開始時刻より後であること
* 同一要員が同一時間帯に複数イベントへ重複割当されないようチェックする

---

### 10.6 顧客登録

* 氏名は必須
* 電話番号は必須
* 電話番号は重複チェックを行う
* 顧客登録時に獲得チャネルを必須とする
* 催事経由の場合、イベントを紐づける

---

### 10.7 アポ登録

* 顧客は必須
* 訪問予定日時は必須
* アポ取得者は必須
* アポ取得組織は必須

---

### 10.8 マエカク結果連絡

* マエカク結果登録後、関係する二次店に結果連絡できること
* 連絡対象の二次店が存在する場合、未連絡状態を管理する
* 連絡済み・確認済みのステータスを管理する

---

### 10.9 商談・クロージング報告

* 商談・クロージング報告権限がない二次店は登録不可
* 顧客は必須
* 商談・クロージングステータスは必須
* 契約ステータスにする場合、契約金額を必須とする
* 失注ステータスにする場合、失注理由を入力可能とする

---

### 10.10 商品・価格マスタ

* 商品名は必須
* 商品カテゴリは必須
* 仕入値は必須
* 二次店卸値は必須
* 適用開始日は必須
* 適用終了日がある場合、適用開始日より後であること
* 価格改定時は既存レコードを直接上書きせず、履歴が残る設計とする

---

### 10.11 契約明細

* 契約に対して1件以上の商品明細を登録できる
* 商品名、数量、仕入値、二次店卸値を保持する
* 契約時点の価格情報を保持する
* 商品マスタ変更後も契約明細の価格は変更されない

---

### 10.12 粗利計算

* 実販売価格は必須
* 商品仕入値合計を契約明細から集計する
* 施工費、その他原価、値引き額を粗利計算に反映する
* 粗利率を自動計算する
* インセンティブ対象粗利を設定できる
* インセンティブ対象粗利を手動調整した場合、変更履歴を残す

---

### 10.13 インセンティブ計算

* 契約が成立していない案件はインセンティブ計算対象外
* キャンセル案件はインセンティブ計算対象外
* 自社開催のみの案件は二次店インセンティブ対象外
* 二次店開催または共同開催案件のみインセンティブ計算対象
* 粗利が0円以下の場合、インセンティブは原則0円
* インセンティブ率が未設定の場合は警告を表示
* 個別調整額を入力できるが、変更履歴を残す

---

## 12. 非機能要件

* ログイン認証を必須とする
* 組織IDベースでデータ閲覧範囲を制御する
* 二次店は他社二次店の情報を閲覧できない
* 二次店は他社の希望店舗、配属状況、成果、インセンティブを閲覧できない
* 自社要員は自分が割り当てられたイベントのみ閲覧できる
* 仕入値は卸業者内部情報とし、二次店には表示しない
* 粗利の内訳は権限に応じて表示制御する
* 個人情報は権限に応じて閲覧制御する
* 監査ログを保持する

---

* 二次店担当者はスマートフォンから報告できること
* 自社現場要員もスマートフォンからシフト確認・報告できること
* イベント報告・アポ顧客登録はスマートフォン操作を前提にする
* 二次店向け画面は入力項目を最小限にする
* 自社要員向け画面はシフト確認・報告に特化する
* 卸業者向け画面は一覧、検索、絞り込み、ステータス更新を重視する

* 一覧画面はページネーションを実装する

## 17. Claude Codeへの実装指示文

以下の仕様書に基づき、太陽光卸事業者と二次店事業者が利用する業務管理Webアプリを開発してください。

本システムは、太陽光パネルの卸業者と営業業務を担う二次店が共同で利用する業務ツールです。

### 重要な業務要件

1. イベントは、最初から確定イベントとして管理するのではなく、まず場所提供元との調整を行い、場所・日程を確定したうえで「イベント候補」として管理してください。
2. 場所提供元対応管理では、場所提供元名、担当者、連絡先、場所名、住所、実施候補日、確定日、契約条件、固定費、成果報酬率、調整ステータスを管理してください。
3. 二次店に対して開催体制決定前に共有する情報は、原則として「場所」と「日にち」のみです。
4. 二次店には、場所提供元との契約内容、固定費、成果報酬率、想定売上、想定粗利、他社二次店の希望状況、他社二次店の配属状況、卸業者内部メモを表示しないでください。
5. 二次店は月単位で、希望する店舗を複数選択して回答できるようにしてください。
6. 二次店の希望収集後、卸業者がどの業者がどの店舗で実施するかを決定します。
7. イベント候補に対して、卸業者は開催体制を「自社開催」「二次店開催」「自社・二次店共同開催」「中止」から選択できるようにしてください。
8. 二次店開催の場合、担当二次店を設定してください。
9. 自社・二次店共同開催の場合、担当二次店と自社要員シフトの両方を設定してください。
10. 自社開催または共同開催の場合、担当する自社要員のシフト管理を行えるようにしてください。
11. 自社開催または共同開催イベントには必要人数を設定できるようにし、必要人数に対してシフト割当人数が不足している場合は警告を表示してください。
12. 同一要員が同じ日時に複数イベントへ重複割当されないようにチェックしてください。
13. 自社要員は、自分に割り当てられたイベントのみ閲覧できるようにしてください。
14. 自社要員は、自分が担当するイベントについて、開始報告、終了報告、成果報告、アポ顧客登録を行えるようにしてください。
15. 二次店はイベント開始報告、終了報告、成果報告、アポ顧客登録を行えるようにしてください。
16. マエカクは二次店のスコープではなく、卸業者側のコール部隊が実施します。
17. マエカク後、卸業者は関係する二次店へマエカク結果を連絡できるようにしてください。
18. 二次店のスコープは「アポ獲得まで」「初回訪問まで」「商談・クロージングまで」の3種類としてください。
19. 「トスアップのみ」は「アポ獲得まで」に含めてください。
20. 「マエカク」「契約後フォロー」は二次店スコープに含めないでください。
21. 商談とクロージングは分けず、「商談・クロージング」として一体で管理してください。
22. 太陽光パネル、蓄電池、パワーコンディショナー、架台、その他部材、セット商品などの商品マスタを管理できるようにしてください。
23. 商品ごとに、メーカー、商品名、型番、容量、単位、仕入値、二次店卸値、希望販売価格を管理してください。
24. 商品価格は変更される可能性があるため、適用開始日、適用終了日、有効フラグを管理してください。
25. 契約時には、商品マスタの価格を契約明細にコピーして保持してください。
26. 商品マスタの価格が変更されても、過去契約の粗利計算結果が変わらないようにしてください。
27. 契約単位で、実販売価格、商品仕入値合計、二次店卸値合計、施工費、その他原価、値引き額、粗利、粗利率、インセンティブ対象粗利を管理してください。
28. インセンティブは売上ではなく、粗利を基準に計算してください。
29. 基本計算式は「インセンティブ額 = インセンティブ対象粗利 × インセンティブ率」としてください。
30. インセンティブ対象粗利は、以下のいずれかを設定できるようにしてください。

    * 案件粗利
    * 二次店卸値合計 - 商品仕入値合計
    * 手動入力
31. 粗利が0円以下の場合、インセンティブは原則0円としてください。
32. キャンセル案件はインセンティブ対象外としてください。
33. 自社開催のみの案件は二次店インセンティブ対象外としてください。
34. 二次店開催または自社・二次店共同開催案件のみ、二次店インセンティブ計算対象としてください。
35. 仕入値は卸業者内部情報として扱い、二次店には原則表示しないでください。
36. 二次店には、自社に関係するインセンティブ対象粗利、インセンティブ率、インセンティブ額のみ確認できるようにしてください。
37. 月次報告機能を実装してください。月次報告では、イベント数、対応店舗数、アポ数、有効アポ数、初回訪問数、商談・クロージング数、契約数、契約金額、粗利、インセンティブ見込み額を自動集計し、主な成果・課題・改善アクション・翌月施策などをコメント入力できるようにしてください。
38. イベント成果は、自社開催分、二次店開催分、共同開催分を分けて集計できるようにしてください。
39. 全体成果として、自社開催分、二次店開催分、共同開催分を合算して確認できるようにしてください。
40. 二次店は自社に関係するデータのみ閲覧可能としてください。他社二次店の希望状況、配属状況、成果、インセンティブは絶対に閲覧できないようにしてください。
41. 自社要員は自分に関係するシフト・イベント・顧客のみ閲覧可能としてください。
42. 組織IDベースでアクセス制御を実装してください。
