## 設計意図メモ

- コール部隊は 1 日に数十件処理する。左ペインに対象一覧 + 右ペインに詳細 + フォームの 2 ペイン構造でキー操作を最小化
- 「次のキューへ」で保存後すぐ次の対象を表示。連続処理に最適化
- スマホでも 2 ペインは断念せず、リスト → 詳細遷移 + 「次のキューへ」で連続性を維持
- 結果セレクトの選択肢 5 つ（承認 / 不在 / 折り返し待ち / キャンセル / 日程変更）を明示。F-035 と一致
