## desktop.png プロンプト

```
Layout: 1440x900 pixels. Top bar (64px): "Solar SaaS" + 右リンク "サインインに戻る"。

中央カード (560x600 px):

### Section 1: ヘッダー
- 見出し "管理者アカウントを作成"
- ステップインジケータ `① 招待コード ✓ ── ② 会社情報 ✓ ── ③ 管理者アカウント`（現在地: ③）

### Section 2: フォーム
- ラベル "管理者メールアドレス" / 下線 / プレースホルダ "admin@company.example"
- グレー注記 "このメールアドレスでサインインします。"
- ラベル "パスワード" / 下線 / 目アイコン
- 強度メーター + "強度: 中"
- グレー注記 "8 文字以上、英大文字・小文字・数字を含めてください。"
- ラベル "パスワード（確認）" / 下線

### Section 3: 同意
- チェックボックス `[ ]` "利用規約 / プライバシーポリシーに同意します"
- チェックボックス `[ ]` "卸業者が定める業務スコープに従って業務を遂行することに同意します"

### Section 4: 2FA 推奨案内
- グレーボックス "次の画面で 2 段階認証の設定をおすすめします（推奨）。"

### Section 5: アクション
- ボタン `[ ← 戻る ]` (左)
- ボタン `[ アカウントを作成 ]`（右、primary）
```

## 設計意図メモ

- 2FA は二次店管理者では任意だが、初回推奨を明示。S-003 への自然な遷移を促す
- スコープ同意のチェックボックスは別立て。卸業者との関係においてスコープが業務上の制約となることを認識してもらう
- 必須ロール（卸業者管理者）と異なり「推奨」表現で柔らかく
