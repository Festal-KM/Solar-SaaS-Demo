# Wireframes — 太陽光卸・二次店営業管理 SaaS

`docs/04-ui-design.md` で定義された全 85 画面（S-001〜S-085）の **ChatGPT 用ワイヤーフレーム生成プロンプト**をここに格納する。

## ディレクトリ構造

```
docs/wireframes/
  README.md                           ← 本ファイル
  S-001-signin/
    prompt.md                         ← ChatGPT に貼り付けるプロンプト
    desktop.png                       ← ChatGPT で生成し運営者が保存
    mobile.png
    empty.png                         ← 該当する場合のみ
    loading.png                       ← 該当する場合のみ
    error.png                         ← 該当する場合のみ
  S-002-2fa-code/
    prompt.md
    ...
  ...
  S-085-audit-saas-admin/
    prompt.md
```

ファイル名は **固定**：
- `desktop.png` — デスクトップ（1440x900）標準ビュー
- `mobile.png` — モバイル（375x812）1 カラム
- `empty.png` — データ未登録時（該当画面のみ）
- `loading.png` — ローディング中（該当画面のみ）
- `error.png` — エラー時（該当画面のみ）

特殊な状態（例: バルク選択時）を加える場合は kebab-case で命名し、該当 `prompt.md` の「想定画像」セクションに必ず列挙する。

## 運用フロー

1. 対象画面のディレクトリへ移動 → `prompt.md` を開く
2. ChatGPT（GPT-4o / 画像生成有効）で新規チャットを開始
3. `prompt.md` の「共通プロンプト」セクションをコピーして貼り付け
4. 続けて生成したいバリアント（`desktop.png` / `mobile.png` / `empty.png` ...）のプロンプトをコピーして同一チャットに貼り付け
5. ChatGPT が生成した PNG をダウンロードし、`docs/wireframes/S-xxx-slug/` 配下に **指定のファイル名で**保存
6. 必要に応じて `prompt.md` を修正して再生成

## 命名規約

- 画面 ID は `docs/04-ui-design.md` §1 と一致させる（S-001〜S-085、欠番なし）
- ディレクトリ名は `S-{id}-{slug}` 形式（slug は kebab-case 英数字、画面の英語的キーワード）
- 例: `S-027-event-decision`, `S-041-contract-detail`, `S-057-field-customer-form`

### 個人情報マスキング

- 卸業者本部 / 二次店本部の顧客系画面では、電話番号は `***-****-1234`（下 4 桁）、住所は `市区町村まで` をデフォルト表示
- 各レコードに鍵アイコン `[🔓]`（権限ありユーザーのみ活性）で `RevealPiiButton` を併設
- 運営者画面は常時マスク固定（鍵アイコンなし）
- 監査ログは常時マスク固定

### スタイル原則（全画像共通）

- **純白黒ワイヤーフレーム**。淡いグレーは「非活性 / プレースホルダ / マスク」表現のみで使用
- ボタン: `[ ボタンラベル ]`（角括弧 + テキスト）
- 入力欄: ラベル上配置、下線 `_______`
- セレクト: `[ ラベル ▾ ]`
- バッジ: `「ステータス名」` または `[ステータス]`
- アバター: `[A]`（一文字プレースホルダ）
- 一覧テーブル: 8〜12 行を実データ風に表示
- カードリスト: 5〜10 件を表示
- アノテーション・ラベル: **すべて日本語**
- 色なし・影なし・角丸なし（鋭利な四角のみ）
- 写真・ロゴ・装飾グラフィックなし
- 情報密度は現実的に。空セクションは空状態バリアント以外では避ける
