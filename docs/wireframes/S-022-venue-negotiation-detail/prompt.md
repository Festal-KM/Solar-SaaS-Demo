## 設計意図メモ

- ステータス遷移を縦タイムラインで可視化。現在地と未到達を視覚的に分離
- 「イベント候補化」ボタンはステータス「確定」時のみ活性。ツールチップで活性条件を提示
- 対応履歴はチャット風コメントスレッド。場所提供元との折衝記録を時系列で
- マスタへのリンクで二重管理を避ける
