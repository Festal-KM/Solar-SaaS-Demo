## 設計意図メモ

- 契約は本ツールの中核。明細 / 粗利 / インセンティブ / 施工 / 申請 / 履歴をタブで統合。1 画面で完結
- SnapshotChip [契約時点の価格] でユーザに「マスタ改定が反映されない」を常時明示。F-041 の重要原則
- キャンセル期限はヘッダー右上でカウントダウン。期限内/期限後で扱いが変わるため最重要 UI
- 月次ロックエラーはモーダル明示 + アンロック導線。F-050 の業務ルール
- 自社案件 vs 共同/二次店案件で表示内容が変わる。ワイヤーでは自社案件で例示しグレー注記で共同時の挙動を示す
